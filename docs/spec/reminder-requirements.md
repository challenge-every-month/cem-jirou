# リマインダー 要件定義書

## 概要

Cloudflare Cron Triggers を使用して、月次チャレンジのライフサイクルに合わせた自動通知を行う。
チャンネルへの一斉通知（コミュニティの場の空気づくり）と、個人 DM による行動促進通知の2種類を提供する。

> **参考**: 元実装（`.legacy/cem-app`）は App Engine Cron + HTTPエンドポイント方式。今回は Cloudflare Cron Triggers の `scheduled` ハンドラーに移行する。

## 通知スケジュール一覧

| タイミング | cron式 (UTC) | 日本時間 | 種別 | 対象 |
|-----------|-------------|---------|------|------|
| 月初 | `0 0 1 * *` | 毎月1日 9:00 | チャンネル投稿 | 全員 |
| 月初（個人） | `0 1 1 * *` | 毎月1日 10:00 | DM | 未設定ユーザー（`personal_reminder=true` のみ） |
| 月中 | `0 0 15 * *` | 毎月15日 9:00 | チャンネル投稿 | 全員 |
| 月末 | `0 0 25 * *` | 毎月25日 9:00 | チャンネル投稿 | 全員 |
| 月末（個人） | `0 1 25 * *` | 毎月25日 10:00 | DM | 未振り返りユーザー（`personal_reminder=true` のみ） |
| 期日接近 | `0 0 * * *` | 毎日 9:00 | DM | due_on 3日以内（`personal_reminder=true` のみ） |

## ユーザストーリー

### ストーリー1: 月初の行動喚起

- **である** コミュニティメンバー **として**
- **私は** 月初にチャンネルで「今月も始まるぞ」という空気を感じたい
- **そうすることで** 月次サイクルに乗って自然と目標設定モードに入れる

### ストーリー2: 未設定者への個別フォロー

- **である** 目標を設定し忘れているメンバー **として**
- **私は** 月初に「まだ設定してないよ」と個別に声をかけてほしい
- **そうすることで** 忙しくても設定のきっかけをもらえる

### ストーリー3: 月中の進捗確認

- **である** コミュニティメンバー **として**
- **私は** 月の半ばに進捗報告を促すメッセージをチャンネルで受け取りたい
- **そうすることで** 折り返し地点として自分の状況を見直せる

### ストーリー4: 期日接近の通知

- **である** due_on を設定したメンバー **として**
- **私は** 期日が近づいたら DM で教えてほしい
- **そうすることで** 設定した期日を見落とさずに行動できる

## 機能要件（EARS記法）

### 月初リマインダー（1日）

- REQ-RMD-001: 毎月1日 9:00 JST に、システムは `SLACK_POST_CHANNEL_ID` に月初メッセージを投稿しなければならない
- REQ-RMD-002: 月初メッセージは `<!channel>` メンションを含み、今月の目標設定・表明を促す内容でなければならない
- REQ-RMD-003: 1月1日の場合、通常の月初メッセージに加えて新年のメッセージを含めなければならない
- REQ-RMD-004: 毎月1日 10:00 JST に、システムは `personal_reminder=true` かつ当月の Project 未作成のユーザーにのみ DM を送信しなければならない

### 月中リマインダー（15日）

- REQ-RMD-011: 毎月15日 9:00 JST に、システムは `SLACK_POST_CHANNEL_ID` に月中メッセージを投稿しなければならない
- REQ-RMD-012: 月中メッセージは進捗報告（`/cem_progress`）を促す内容でなければならない

### 月末リマインダー（25日）

- REQ-RMD-021: 毎月25日 9:00 JST に、システムは `SLACK_POST_CHANNEL_ID` に月末メッセージを投稿しなければならない
- REQ-RMD-022: 月末メッセージは振り返り（`/cem_review`）を促す内容でなければならない
- REQ-RMD-023: 12月25日の場合、通常の月末メッセージに加えて年末のメッセージを含めなければならない
- REQ-RMD-024: 毎月25日 10:00 JST に、システムは `personal_reminder=true` かつ当月 Project が未 `reviewed` のユーザーにのみ DM を送信しなければならない

### 期日接近リマインダー（毎日）

- REQ-RMD-031: システムは毎日 9:00 JST に、`personal_reminder=true` かつ `due_on` が翌日から3日以内の Challenge を持つユーザーに DM を送信しなければならない
- REQ-RMD-032: 期日接近 DM には対象 Challenge の名前・期日・所属 Project を含めなければならない
- REQ-RMD-033: `completed` または `incompleted` 状態の Challenge は期日接近通知の対象から除外しなければならない

### 条件付き要件

- REQ-RMD-101: 個人 DM の対象ユーザーが存在しない場合、システムは DM 処理をスキップしなければならない（エラーにしない）
- REQ-RMD-102: チャンネル投稿が失敗した場合、システムはエラーをログに記録し処理を継続しなければならない
- REQ-RMD-103: 個人 DM 送信が一部ユーザーで失敗した場合、残りのユーザーへの送信を継続しなければならない

### 制約要件

- REQ-RMD-401: Cron スケジュールは `wrangler.toml` の `[triggers]` セクションで定義しなければならない
- REQ-RMD-402: Cron ハンドラーは Worker の `scheduled` イベントとして実装しなければならない（HTTP エンドポイント方式は採用しない）
- REQ-RMD-403: 全ユーザースキャンは Cloudflare Workers の CPU 時間制限（30秒）以内に完了しなければならない
- REQ-RMD-404: チャンネル投稿・DM 送信の宛先設定は環境変数で管理し、コードにハードコードしてはならない

## wrangler.toml Cron 設定

```toml
[triggers]
crons = [
  "0 0 1 * *",   # 月初チャンネル投稿 (JST 9:00)
  "0 1 1 * *",   # 月初個人DM (JST 10:00)
  "0 0 15 * *",  # 月中チャンネル投稿 (JST 9:00)
  "0 0 25 * *",  # 月末チャンネル投稿 (JST 9:00)
  "0 1 25 * *",  # 月末個人DM (JST 10:00)
  "0 0 * * *",   # 期日接近チェック (JST 9:00)
]
```

## 通知メッセージ仕様

### 月初チャンネル（通常月）
```
<!channel>
新しい月の始まりです！今月も目標を立てて一歩一歩行きましょう 💪
Let's Challenge！
```

### 月初チャンネル（1月）
```
<!channel>
謹賀新年！新しい年の始まりです 🎍
今年はどんな年にしましょうか？今月の目標を立てて、Let's Challenge！
```

### 月初個人 DM（未設定者）
```
今月の目標をまだ設定していません 📝
/cem_new で今月のチャレンジを登録しましょう！
```

### 月中チャンネル
```
<!channel>
月の折り返し地点です 📊 進捗どうですか？
/cem_progress で現状を報告してみましょう！
```

### 月末チャンネル（通常月）
```
<!channel>
今月もお疲れさまでした 🎉
振り返りをして今月を締めくくりましょう！ /cem_review
```

### 月末個人 DM（未振り返り）
```
今月の振り返りがまだです ✍️
/cem_review で達成・未達成を確定しましょう！
```

### 期日接近 DM
```
期日が近づいているチャレンジがあります ⏰

• {project_title} / {challenge_name}（期日: {due_on}）
```

## 非機能要件

### パフォーマンス

- NFR-RMD-001: 全ユーザースキャン処理は Workers の CPU 時間制限（30秒）以内に完了しなければならない
- NFR-RMD-002: ユーザー数が増加した場合、Slack API のレート制限（Tier 3: 50req/min）を考慮したバッチ送信を行わなければならない

### 信頼性

- NFR-RMD-101: Cron ハンドラーは冪等に実装し、二重実行されても重複通知が発生しないようにしなければならない

## Edge ケース

- EDGE-RMD-001: 登録ユーザーが0人の場合、DM スキャンは何もせず正常終了する
- EDGE-RMD-002: ユーザーが Bot を DM ブロックしている場合、該当ユーザーの DM 送信エラーを無視して継続する
- EDGE-RMD-003: Cron 実行時に D1 が一時的に利用不可の場合、エラーをログに記録して次回実行を待つ（リトライしない）

## 受け入れ基準

- [ ] `wrangler.toml` に6つの Cron スケジュールが定義されていること
- [ ] 月初（1日）にチャンネルへの投稿と、未設定ユーザーへの DM が実行されること
- [ ] 月中（15日）にチャンネルへの投稿が実行されること
- [ ] 月末（25日）にチャンネルへの投稿と、未振り返りユーザーへの DM が実行されること
- [ ] 期日3日前の Challenge を持つユーザーに毎日 DM が送られること
- [ ] `completed` / `incompleted` の Challenge が期日通知対象から除外されること
- [ ] 1月1日に新年メッセージが含まれること
- [ ] Cron 二重実行で重複通知が発生しないこと
