# App Home Tab 要件定義書

## 概要

Slack の App Home タブをメインインターフェースとして使用する。
各ユーザーが自分の月次チャレンジを登録・確認・操作できるダッシュボードを提供する。
スラッシュコマンド（`/cem_new` 等）は引き続き使用可能なショートカットとして残す。

Web インターフェースは現フェーズではスコープ外とし、App Home で履歴閲覧まで完結させる。

## ユーザストーリー

### ストーリー1: ダッシュボードの確認

- **である** コミュニティメンバー **として**
- **私は** Slack の CEM アプリを開いたときに今月のチャレンジ一覧をすぐ見たい
- **そうすることで** コマンドを打たずに現在の状況を把握できる

### ストーリー2: ボタン操作での目標管理

- **である** コミュニティメンバー **として**
- **私は** App Home 上のボタンだけで追加・編集・削除・完了をしたい
- **そうすることで** コマンドを覚えなくても直感的に操作できる

### ストーリー3: 過去月の履歴閲覧

- **である** コミュニティメンバー **として**
- **私は** App Home 上で前月・前々月の記録を見たい
- **そうすることで** 自分の達成履歴や失敗も振り返れる

## 画面構成（Block Kit レイアウト）

```
┌──────────────────────────────────────────────┐
│ 👋 こんにちは、@aqui!                          │
│ 2026年3月 のチャレンジ    [← 前月]  [次月 →] │
├──────────────────────────────────────────────┤
│ 📦 英語学習                      [published] │
│   ✅ Anki 30分                               │
│   🔵 Podcast 聴く          期日: 3/20        │
│   [+ チャレンジ追加]  [✏️ 編集]  [🗑️ 削除] │
├──────────────────────────────────────────────┤
│ 📦 その他                           [draft]  │
│   🔵 筋トレ                                  │
│   🔵 本読む                                  │
│   [+ チャレンジ追加]  [✏️ 編集]             │
├──────────────────────────────────────────────┤
│      [＋ プロジェクト追加]                    │
│      [📣 今月を宣言する]                      │
└──────────────────────────────────────────────┘
```

**Challenge ステータスアイコン:**

| アイコン | status | 意味 |
|---------|--------|------|
| ⚪ | `draft` | 未公開 |
| 🔴 | `not_started` | 未着手（公開済） |
| 🔵 | `in_progress` | 進行中 |
| ✅ | `completed` | 達成 |
| ❌ | `incompleted` | 未達成（確定） |

## 機能要件（EARS記法）

### 通常要件

- REQ-HOME-001: システムは `app_home_opened` イベント受信時に、該当ユーザーの App Home を描画しなければならない
- REQ-HOME-002: システムは App Home に、`viewed_year` / `viewed_month` に対応する Project 一覧（Challenge を含む）を表示しなければならない
- REQ-HOME-003: システムは App Home に [← 前月] / [次月 →] ボタンを表示し、押下時に `viewed_year` / `viewed_month` を更新して再描画しなければならない
- REQ-HOME-004: システムは [＋ プロジェクト追加] ボタン押下時に、Project / Challenge 登録モーダルを開かなければならない
- REQ-HOME-005: システムは [+ チャレンジ追加] ボタン押下時に、該当 Project に Challenge を追加するモーダルを開かなければならない
- REQ-HOME-006: システムは [✏️ 編集] ボタン押下時に、Multi-step モーダル（Project 選択 → 編集フォーム）を開かなければならない
- REQ-HOME-007: システムは [🗑️ 削除] ボタン押下時に確認ダイアログを表示し、承認後に Project と配下 Challenge を削除して再描画しなければならない
- REQ-HOME-008: システムは [📣 今月を宣言する] ボタン押下時に、`draft` 状態の Project を `published` に更新し、指定チャンネルに宣言メッセージを投稿しなければならない
- REQ-HOME-009: システムは `not_started` / `in_progress` の Challenge に [未着手] / [進行中] / [✅済] ボタンを表示し、押下で status を更新して再描画しなければならない
- REQ-HOME-010: システムは各 Challenge の ⋮ メニューから「コメントを追加」を選択できるようにし、押下でコメント入力のミニモーダルを開かなければならない

### 条件付き要件

- REQ-HOME-101: `app_home_opened` 時に `user_preferences` が存在しない場合、システムは Lazy Provision を実行してからホームを描画しなければならない
- REQ-HOME-102: `viewed_year` / `viewed_month` が未設定の場合、システムは現在の年月をデフォルトとして使用しなければならない
- REQ-HOME-103: 表示中の月に Project が存在しない場合、システムは「まだチャレンジがありません」という空状態メッセージを表示しなければならない
- REQ-HOME-104: `viewed_month` が現在月より未来の場合、[次月 →] ボタンを非活性にしなければならない
- REQ-HOME-105: `reviewed` 状態の Project には [✏️ 編集] / [🗑️ 削除] ボタンを表示してはならない
- REQ-HOME-106: Project の status が `draft` の場合のみ [📣 今月を宣言する] ボタンを表示しなければならない
- REQ-HOME-107: 全 Project が `published` または `reviewed` 状態の場合、[📣 今月を宣言する] ボタンを非表示にしなければならない

### 状態要件

- REQ-HOME-201: `published` 状態の Project の Challenge が全て `completed` または `incompleted` に確定した場合、[振り返りを完了する] ボタンを表示し、Project を `reviewed` に更新できなければならない

### 制約要件

- REQ-HOME-401: App Home の描画は `app_home_opened` イベント受信から 3 秒以内に完了しなければならない
- REQ-HOME-402: ボタン操作後の再描画は操作完了から 1 秒以内に反映されなければならない
- REQ-HOME-403: App Home は各ユーザー固有のビューを表示し、他ユーザーのデータを表示してはならない

## `user_preferences` への追加カラム

App Home の月ナビ状態管理のため、以下を追加する：

```
user_preferences
+ viewed_year  INTEGER  DEFAULT NULL  ← NULL の場合は現在年を使用
+ viewed_month INTEGER  DEFAULT NULL  ← NULL の場合は現在月を使用
```

## 非機能要件

### パフォーマンス

- NFR-HOME-001: `app_home_opened` から描画完了まで 1 秒以内（通常時）
- NFR-HOME-002: ボタン操作（Challenge ステータス更新等）は 500ms 以内に反映

### 保守性

- NFR-HOME-201: App Home のビュー構築ロジックは `src/views/home.ts` に集約しなければならない
- NFR-HOME-202: Block Kit の各セクションはコンポーネント関数として分離しなければならない

## Edge ケース

- EDGE-HOME-001: Block Kit の最大ブロック数（100件）を超える場合、Challenge 表示を折りたたむかページネーションを設ける
- EDGE-HOME-002: `app_home_opened` が短時間に連続発火した場合、最後のイベントのみを処理する（debounce）
- EDGE-HOME-003: App Home 描画中にエラーが発生した場合、エラーメッセージのみを含む最小限のビューを表示する

## 受け入れ基準

### 機能テスト

- [ ] アプリを開くと今月の Project / Challenge 一覧が表示されること
- [ ] [← 前月] で過去月に移動でき、その月のデータが表示されること
- [ ] 現在月表示時に [次月 →] ボタンが非活性であること
- [ ] [＋ プロジェクト追加] でモーダルが開き、登録後に Home が更新されること
- [ ] [✅] ボタンで Challenge が `completed` になりホームが即時更新されること
- [ ] [📣 今月を宣言する] 押下でチャンネルに宣言メッセージが投稿されること
- [ ] Project が存在しない月に空状態メッセージが表示されること
- [ ] `reviewed` の Project に編集・削除ボタンが表示されないこと
- [ ] 初回アクセス（未登録）で Lazy Provision が実行されホームが表示されること

### 非機能テスト

- [ ] `app_home_opened` から 1 秒以内にホームが描画されること
- [ ] 自分のデータのみが表示され、他ユーザーのデータが混入しないこと
